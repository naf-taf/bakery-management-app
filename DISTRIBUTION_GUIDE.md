# Руководство по распространению Bakery Management App

## Обзор

Это руководство описывает процесс подготовки и распространения desktop-приложения Bakery Management среди конечных пользователей.

**Функциональность приложения:**

- Управление ингредиентами с указанием стоимости в белорусских рублях (BYN)
- Создание рецептов выпечки
- Планирование производства
- Генерация списков замеса с расчетом стоимости

**Валюта:** Приложение работает с белорусскими рублями (BYN) как основной валютой для расчета стоимости ингредиентов и готовой продукции.

## Системные требования

- **Операционная система**: Windows 7 SP1 / 8 / 10 / 11
- **Процессор**: 1 GHz или выше
- **Оперативная память**: 512 MB минимум
- **Место на диске**: 100 MB для установки
- **Дополнительно**: .NET Framework 4.5+ (для установщика)

## Шаг 2: Сборка приложения

### Сборка renderer-приложения

```bash
# Из корневой папки
npm run build
```

Эта команда:

- Компилирует renderer-приложение (Vite) в production-режиме
- Оптимизирует код и ресурсы
- Создает папку `src/renderer/build/`

### Создание дистрибутива

```bash
# Из корневой папки
npm run dist
```

Эта команда:

- Выполняет сборку renderer (если не была выполнена)
- Упаковывает Electron-приложение
- Создает установщик и портативную версию
- Выводит файлы в папку `dist/`

## Шаг 4: Подготовка к распространению

### Вариант 1: Установщик Windows (рекомендуется)

**Файл:** `Bakery Management Setup X.X.X.exe`

**Преимущества:**

- Полная интеграция с системой

- Удаление через "Программы и компоненты"
- Автоматические обновления (если настроены)

**Инструкции для пользователей:**

1. Скачайте файл установщика
2. Запустите файл от имени администратора
3. Следуйте инструкциям мастера установки
4. Выберите папку установки (по умолчанию Program Files)
5. После установки запустите приложение из меню "Пуск"

### Вариант 2: Портативная версия

**Файл:** `Bakery Management X.X.X.exe`

**Преимущества:**

- Не требует установки
- Можно запускать с USB-накопителя
- Не оставляет следов в системе
- Легко переносить между компьютерами

**Инструкции для пользователей:**

1. Скачайте exe-файл
2. Разместите в удобной папке
3. Запустите файл двойным щелчком
4. При необходимости создайте ярлык на рабочем столе

## Шаг 10: Обновление приложения

### Процесс выпуска обновления

1. Внесите изменения в код
2. Выберите новую версию по SemVer (`MAJOR.MINOR.PATCH`), например `1.0.4`
3. Обновите версии в обоих файлах: `package.json` и `src/renderer/package.json`
4. Протестируйте изменения (`npm run build`, `npm run dist`)
5. Закоммитьте изменения
6. Создайте тег версии формата `vX.Y.Z`
7. Отправьте ветку и тег в репозиторий
8. GitHub Actions автоматически соберет релиз и опубликует артефакты

Пример:

```bash
npm version 1.0.4 --no-git-tag-version
npm version 1.0.4 --no-git-tag-version --prefix src/renderer

npm run build
npm run dist

git add .
git commit -m "chore: release 1.0.4"
git tag -a v1.0.4 -m "Release v1.0.4"
git push origin main
git push origin v1.0.4
```

Workflow `.github/workflows/release.yml` срабатывает при push тега `v*.*.*`, публикует GitHub Release и прикрепляет файлы из `dist/`.

Правило выбора версии (SemVer):

- PATCH (`1.0.3 -> 1.0.4`) для исправлений без изменения API/поведения для пользователей.
- MINOR (`1.0.3 -> 1.1.0`) для новых возможностей с обратной совместимостью.
- MAJOR (`1.0.3 -> 2.0.0`) для несовместимых изменений.

Краткий чеклист релиза (copy-paste):

```bash
# Укажите новую версию один раз
set VERSION=1.0.4

# Обновите версии в обоих package.json
npm version %VERSION% --no-git-tag-version
npm version %VERSION% --no-git-tag-version --prefix src/renderer

# Проверка сборки
npm run build
npm run dist

# Коммит и тег
git add .
git commit -m "chore: release %VERSION%"
git tag -a v%VERSION% -m "Release v%VERSION%"

# Публикация ветки и тега
git push origin main
git push origin v%VERSION%
```
